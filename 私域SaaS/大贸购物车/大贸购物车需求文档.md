# 大贸购物车需求文档

## 1. 业务背景

### 1.1 需求描述
因存在多个商品需一起结算享受优惠，平台需新增购物车能力。商品类型包含普通商品、大贸跨境商品、直邮商品，本期做通用购物车框架，但仅支持普通商品加入购物车并批量结算。

批量结算后，待支付阶段展示整单；用户支付后按供应商库位拆分为子单，并按既有线上逻辑执行推单、发货、售后。

### 1.3 目标结果
1. 用户可在小程序商品详情页、选规格弹窗将普通商品加入购物车。
2. 用户可在小程序购物车勾选多个普通商品并批量提交订单。
3. 待支付订单展示整单，支付后按子单展示。
4. PC 订单中心可查看待支付整单、支付后子单、多商品订单明细、推送状态、分账信息。
5. 商品批次支持查看是否允许加入购物车，并展示供应商库位编码。

### 1.4 本期范围
1. PC 后台：业务配置增加购物车配置。
2. PC 后台：商品中心-商品管理列表与批次弹窗。
3. PC 后台：订单中心-订单管理列表、订单详情、订单日志、分账信息。
4. 小程序：商品详情、选规格弹窗、购物车、确认订单、我的订单、订单详情。
5. 普通商品支持加入购物车、批量结算。
6. 跨境、直邮商品仅保留类型框架，本期不支持加购和批量结算。

### 1.5 本期不做
1. 跨境商品购物车加购和批量结算。
2. 直邮商品购物车加购和批量结算。
3. 购物车内修改商品规格。
4. 支付、发货、售后核心链路重构。

## 2. 关键业务规则

### 2.1 PC 后台-业务配置

#### 2.1.1 页面入口
1. 菜单位置：系统配置 / 业务配置。
2. 在订单配置下新增「购物车配置」。

#### 2.1.2 字段定义
1. 购物车开关：单选组，必填。
   - 可选值：开启、关闭。
   - 默认值：关闭。
   - 说明：开启后，商品详情与选规格弹窗可展示「加入购物车」。
2. 商品加购范围：复选组，购物车开关为开启时展示。
   - 可选值：普通商品。
   - 默认值：无默认勾选。
   - 本期仅允许选择普通商品。
3. 结算限制：本期不展示。

#### 2.1.3 联动逻辑
1. 当购物车开关为关闭时：
   - 商品详情页「加入购物车」不可用。
   - 选规格弹窗不允许执行加入购物车。
2. 当购物车开关为开启，且商品加购范围勾选普通商品时：
   - 普通商品商品详情展示「加入购物车」。
   - 普通商品选规格弹窗展示「加入购物车」和「立即购买」。

### 2.2 PC 后台-商品中心-商品管理

#### 2.2.1 页面入口
1. 菜单位置：商品中心 / 商品管理。
2. 页面支持商品列表查询、状态 Tab、导出、新增、导入修改等入口。

#### 2.2.2 搜索字段
1. 商品名称：文本输入，模糊匹配商品名称。
2. 商品类型：下拉单选，支持全部、普通商品、跨境商品、直邮商品。
3. 商品品牌：下拉单选，按品牌筛选。
4. 所属 SPU：下拉单选，按 SPU 筛选。

#### 2.2.3 列表字段
1. 商品：展示商品图片、商品名称、商品 ID。
2. SPU：展示商品所属 SPU。
3. 品牌：展示商品品牌。
4. 分类：展示商品分类。
5. 商品来源：展示 API 对接等来源。
6. 发货仓：展示商品发货仓。
7. 售价：金额，单位元。
8. API 商品编码：蓝色文字按钮，点击打开批次弹窗。
9. 总库存：整数。
10. 访客数：整数。
11. 访问量：整数。
12. 销量：整数。
13. 虚拟销量：整数。
14. 创建：展示创建人和创建时间。
15. 操作：编辑、预览、复制。

#### 2.2.4 批次弹窗
1. 触发方式：点击列表「API 商品编码」。
2. 弹窗名称：查看批次。
3. 字段：
   - 批次 ID
   - 供应商编码
   - 供应商名称
   - 供应商库位编码
   - 中台批次号
   - 推送系统
   - 全球云链推送系统编码
   - 中台状态
   - 规格数量
   - 总规格毛重(kg)
   - 商品单价
   - 建议零售单价
   - 最低零售单单价
   - 商品总价
   - 起购量
   - 每单限购量
   - 允许加入购物车
   - 商城状态
   - 有效期
   - 操作
4. 操作列：仅展示上架/下架，不展示编辑。
5. 供应商库位编码字段来源：批次同步字段 `supplierWareSeatCode`。

### 2.3 PC 后台-订单中心-订单管理列表

#### 2.3.1 页面入口
1. 菜单位置：订单中心 / 订单管理。
2. 页面用于查看普通订单、跨境订单、直邮订单。

#### 2.3.2 状态 Tab
1. 全部订单
2. 待付款
3. 待发货
4. 已发货
5. 已完成
6. 已关闭

#### 2.3.3 搜索字段
1. 关键词：文本输入，模糊匹配订单编号、收货人、商品名称、收货手机号、会员编号、会员昵称。
2. 订单类型：下拉单选，支持全部、普通订单、跨境订单、直邮订单。
3. 推送状态：下拉单选，支持全部、推送成功、推送失败、不需要推送。

#### 2.3.4 列表字段
1. 商品：展示商品图片、商品名称、规格、商品 ID。
2. 单价：按商品维度展示，单位元。
3. 数量：按商品维度展示。
4. 优惠：按商品维度展示。
5. 售后状态：按商品维度展示；未申请不展示，显示为空。
6. 发货仓：按订单维度展示，多个商品时合并单元格。
7. 用户：按订单维度展示，包含用户 ID、昵称、收货人、手机号。
8. 订单状态：按订单维度展示。
9. 推送状态：按订单维度展示。

#### 2.3.5 多商品订单展示
1. 一个订单包含 2 个及以上商品时：
   - 商品、单价、数量、优惠、售后状态按商品逐行展示。
   - 发货仓、用户、订单状态、推送状态按订单合并展示。
2. 待付款多商品订单展示整单，不拆子单。
3. 支付后按供应商库位拆成子单展示。

#### 2.3.6 操作入口
1. 详情：所有状态订单展示，点击进入订单详情。
2. 取消支付：仅待付款订单展示。
   - 点击后弹出二次确认。
   - 确认后订单状态变为已关闭。
   - 推送状态变为不需要推送。
3. 发货：待发货订单展示，点击后订单状态变为已发货。
4. 完成：已发货订单展示，点击后订单状态变为已完成。
5. 重新推送：推送状态为推送失败时展示。
   - 点击后推送状态变为推送成功。

#### 2.3.7 推送状态规则
1. 推送成功：订单已完成拆单并成功推送中台。
2. 推送失败：订单拆单后推送中台失败，可重新推送。
3. 不需要推送：待付款订单、已关闭订单不推送中台。

### 2.6 小程序-商品详情

#### 2.6.1 页面展示
1. 底部展示客服、购物车、收藏、分享。
2. 底部操作按钮展示加入购物车、立即购买。
3. 客服、购物车、收藏、分享文字不使用斜体。

#### 2.6.2 加入购物车
1. 点击加入购物车打开选规格弹窗。
2. 若后台购物车开关关闭，或商品类型不在加购范围内，提示不可加购。

4. 点击加入购物车后：
   - 若购物车已有相同商品，数量累加。
   - 若购物车无相同商品，新增购物车商品。
   - 加购成功后跳转购物车。

#### 2.6.3 立即购买
1. 点击立即购买打开选规格弹窗。
2. 选规格弹窗中点击立即购买后进入确认订单。
3. 确认订单商品仅展示当前立即购买商品和所选数量。

### 2.7 小程序-购物车

#### 2.7.1 页面展示
1. 顶部 Tab 仅展示全部、普通。
2. 商品列表仅展示普通商品。
3. 页面至少展示 4 个普通商品样式：
   - 无活动商品。
   - 折扣立减商品。
   - 秒杀商品。
   - 左滑可收藏、删除商品。
4. 商品可勾选、取消勾选、调整数量。
5. 底部展示全选、合计金额、结算按钮。
6. 全选按钮和全选文字需垂直对齐。

#### 2.7.2 左滑操作
1. 部分商品展示无收藏、删除操作区的普通样式。
2. 左滑商品展示收藏、删除操作区。
3. 收藏：
   - 点击后切换收藏状态。
   - 文案在收藏、已收藏之间切换。
4. 删除：
   - 点击后弹出二次确认。
   - 确认后从购物车移除商品。

#### 2.7.3 价格展示规则
1. 无活动：
   - 展示售价。
2. 折扣立减活动：
   - 展示减后价/折后价。
   - 展示活动标签「折扣立减」。
   - 展示原售价，原售价使用删除线。
3. 秒杀活动：
   - 展示秒杀价。
   - 展示活动标签「秒杀」。
   - 展示原售价，原售价使用删除线。

#### 2.7.4 结算
1. 点击结算时，取购物车当前已勾选的普通商品进入确认订单。
2. 未勾选可结算商品时，Toast 提示：请先选择可结算的普通商品。
3. 购物车合计金额按活动价 * 数量计算。

7. 商品数量展示为 `xN`。
8. 购买须知仅展示《大贸下单协议2》。

#### 2.8.3 商品价格展示规则
1. 无活动：
   - 展示售价。

   - 不展示原售价。

   - 展示秒杀价。

5. 积分抵扣：按订单展示。
6. 订单备注：默认无备注。
7. 不展示税费。
8. 底部总计按确认订单当前商品动态计算。

1. 点击提交订单后，按确认订单当前商品生成待付款订单。
2. 若来源为购物车，则订单商品为购物车已勾选商品。

4. 订单生成后跳转我的订单，待付款展示整单。

## 3. 计算规则

### 3.1 购物车金额
1. 单商品小计 = 商品当前展示价 * 商品数量。
2. 购物车合计 = 已勾选普通商品小计之和。
3. 不可结算商品不计入合计。

### 3.2 确认订单金额
1. 商品原价 = 商品原售价 * 数量汇总。
2. 活动价合计 = 商品当前结算价 * 数量汇总。
3. 活动优惠 = 商品原价 - 活动价合计。
4. 订单实付金额 = 活动价合计 + 运费 - 积分抵扣 - 优惠券抵扣。
5. 本期确认订单不展示税费。

### 3.3 PC 订单详情金额
1. 商品实付总价 = 商品原总价 - 商品优惠。
2. 订单实付金额 = 商品原总价 - 商品优惠 - 积分抵扣 + 商品税费 + 订单运费。
3. 金额单位为元，展示保留 2 位小数。

## 4. 状态与异常

### 4.1 订单状态
1. 待付款
2. 待发货
3. 已发货
4. 已完成
5. 已关闭

### 4.2 推送状态
1. 推送成功
2. 推送失败
3. 不需要推送

### 4.3 状态-操作矩阵
| 订单状态 | PC 可执行操作 | 小程序可执行操作 | 推送状态规则 |
| --- | --- | --- | --- |
| 待付款 | 详情、取消支付 | 去支付 | 不需要推送 |
| 待发货 | 详情、发货 | 申请售后 | 推送成功 / 推送失败 |
| 已发货 | 详情、完成 | 查看物流、确认收货 | 推送成功 |
| 已完成 | 详情 | 查看物流、去评价 | 推送成功 |
| 已关闭 | 详情 | - | 不需要推送 |

### 4.4 状态流转
1. 提交订单：生成待付款整单。
2. 取消支付：待付款 -> 已关闭。
3. 支付成功：待付款整单 -> 待发货子单。
4. 商家发货：待发货 -> 已发货。
5. 确认收货：已发货 -> 已完成。
6. 重新推送：推送失败 -> 推送成功。

### 4.5 异常处理规则
1. 购物车开关关闭时，加入购物车不可用。
2. 商品类型不在加购范围内时，不允许加入购物车。
3. 未勾选可结算普通商品时，不允许进入确认订单。
4. 删除购物车商品需二次确认。
5. 取消支付需二次确认。
6. 推送失败时展示重新推送入口。

## 5. 衍生需求与默认授权项

### 5.1 导出
1. 订单管理列表支持导出。
2. 商品管理列表支持导出。
3. 导出内容需与当前筛选条件联动。

### 5.2 日志
1. 订单生成需记录订单日志。
2. 取消支付需记录订单日志。
3. 商家发货需记录订单日志。
4. 确认收货需记录订单日志。
5. 重新推送需记录推送操作日志。

### 5.3 ID 规则
1. 主单编码：批量结算生成的整单编码。
2. 子单编码：支付后按供应商库位拆分生成，建议格式为主单编码 + 子单序号。
3. 交易主单号：与主单支付交易关联。
4. 交易子单号：与子单履约关联。

### 5.4 端能力限制
1. 本期仅小程序端支持购物车前台能力。
2. PC 后台仅提供配置、商品批次、订单管理、订单详情等运营能力。
3. 跨境、直邮购物车能力后续扩展。
