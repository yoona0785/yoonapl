# 需求模版

---

## 1. 业务背景

跨境商品、直邮商品存在多个商品一起结算享受优惠的业务场景，平台需在已有大贸购物车能力上扩展跨境购物车能力。商品类型包含普通商品、跨境商品、直邮商品，三类商品均可加入购物车，但不同商品类型需分开批量结算。

用户从购物车批量提交订单后，待支付阶段展示整单；用户支付后按商品、数量拆单。跨境订单一种商品一个单，一个单最多 6 个，拆单后按既有线上逻辑执行推单、发货、售后。

### 1.1 需求描述
1. 小程序商品详情页与选规格弹窗支持按后台配置展示或隐藏「加入购物车」入口。
2. 小程序购物车支持普通、跨境、直邮商品分组展示、筛选、全选、批量结算和批量管理。
3. PC 后台支持购物车配置、商品批次查看、订单列表、订单详情、订单日志、分账信息查看。
4. 待支付订单展示整单，支付后展示拆分后的子单。

### 1.3 目标结果
1. 用户可在小程序商品详情页、选规格弹窗将跨境商品、直邮商品加入购物车。
2. 用户可在小程序购物车按商品类型勾选多个商品并批量提交订单。
3. 普通、跨境、直邮商品可同时存在购物车，但批量结算时必须按类型分开提交。
4. 待支付订单展示整单，支付后按子单展示。
5. PC 订单中心可查看待支付整单、支付后子单、多商品订单明细、推送状态、分账信息。

### 1.4 本期范围
- PC 后台：系统配置 / 业务配置，增加购物车配置。
- PC 后台：商品中心 / 商品管理，支持商品列表查询与批次查看弹窗。
- PC 后台：订单中心 / 订单管理，支持订单列表、订单详情、订单日志、分账信息。
- 小程序：商品详情、选规格弹窗、购物车、确认订单、我的订单、订单详情。
- 普通商品、跨境商品、直邮商品支持加入购物车。
- 小程序购物车支持全部 / 大贸 / 跨境 / 直邮筛选。
- 小程序购物车支持分组全选、底部全选、批量结算、批量管理删除。
- 小程序购物车商品排序：先按商品类型，再按加购时间降序排列。

### 1.5 本期不做
- 支付、发货、售后核心链路重构。
- 优惠、税费、运费、分账的真实后端计算重构。
- 商品详情页完整商品信息装修能力重构。

## 2. 关键业务规则

### 4.1 唯一性 / 并发性
- 同一用户、同一商品、同一规格重复加入购物车时，不新增重复行，合并数量，并更新加购时间。
- 购物车商品排序规则固定为：商品类型排序优先，类型内按加购时间倒序。
- 商品类型排序：普通商品（大贸）→ 跨境商品 → 直邮商品。
- 购物车结算提交时需校验商品最新状态、库存、规格有效性。若提交时状态变化，以提交校验结果为准。
- 用户在多个端同时操作购物车时，以最后一次成功保存的购物车数据为准；涉及支付提交时以后端订单创建结果为准。

### 4.2 字段定义
- 购物车开关：控制小程序商品详情页、选规格弹窗是否展示「加入购物车」入口。
  - 是否必填：是。
  - 默认值：关闭。
  - 可选值：开启、关闭。
  - 不允许：关闭时仍展示加入购物车入口。
- 商品加购范围：控制哪些商品类型允许加入购物车。
  - 是否必填：购物车开关开启时必填。
  - 默认值：无默认勾选。
  - 可选值：普通商品、跨境商品、直邮商品。
  - 不允许：未勾选的商品类型展示加入购物车入口。
- 商品类型：购物车分组和结算校验依据。
  - 可选值：普通商品、跨境商品、直邮商品。
  - 页面展示：小程序购物车中普通商品展示为「大贸」。
- 商品规格：用户在选规格弹窗选择的可购买 SKU。
  - 是否必填：是。
  - 不允许：选择库存不足、下架、失效规格加入购物车。
- 加购时间：商品加入购物车或重复加购合并数量时记录的时间。
  - 用途：同类型内购物车排序。
- 订单类型：订单列表筛选与展示字段。
  - 可选值：普通订单、跨境订单、直邮订单。
- 推送状态：订单推送后端系统状态。
  - 可选值：不需要推送、推送成功、推送失败。

### 4.3 计算规则
- 购物车合计金额：当前筛选范围内已勾选且可结算商品的 `商品单价 × 数量` 汇总。
- 购物车结算数量：当前筛选范围内已勾选且可结算商品行数。
- 购物车全选：
  - 底部全选：选择当前筛选范围内所有可结算商品。
  - 分组全选：选择当前分组内所有可结算商品。
  - 失效商品、不可结算商品不参与全选。
- 待支付整单金额：按确认订单页展示的商品金额、活动优惠、积分抵扣、运费等汇总展示；原型中为模拟计算。
- 跨境支付后拆单：
  - 按商品、数量拆单。
  - 一种商品一个单。
  - 一个子单最多 6 个。
  - 同一商品数量超过 6 个时，按 6 个一组继续拆分多个子单。
- 普通商品、直邮商品支付后沿用既有线上拆单、推单、发货、售后逻辑；原型中按订单类型生成模拟子单。

### 4.4 关键联动逻辑
- PC 后台购物车开关关闭：
  - 小程序商品详情页不展示「加入购物车」按钮。
  - 小程序选规格弹窗不展示「加入购物车」按钮。
  - 用户仍可通过「立即购买」进入确认订单。
- PC 后台购物车开关开启，但商品类型未勾选加购范围：
  - 当前类型商品不展示「加入购物车」按钮。
  - 其他已勾选类型按配置展示。
- 选规格弹窗：
  - 用户点击可购买规格后，商品名称、价格、已选规格、库存信息同步刷新。
  - 点击不可购买规格时，提示「该规格库存不足，请选择可购买规格」。
  - 加入购物车时按所选规格写入购物车；同商品同规格合并数量。
- 小程序购物车筛选：
  - 点击全部：展示大贸、跨境、直邮所有分组。
  - 点击大贸：仅展示普通商品分组。
  - 点击跨境：仅展示跨境商品分组。
  - 点击直邮：仅展示直邮商品分组。
- 小程序购物车结算：
  - 当前筛选范围内已勾选商品可进入确认订单。
  - 若勾选商品包含多个商品类型，提示需分开结算，不允许提交。
- 批量管理：
  - 进入管理态后，底部主操作由「结算」切换为「删除」。
  - 选择商品后可批量删除，删除需二次确认。

### 4.5 异常处理规则
- 未选择商品点击结算：提示「请先选择可结算商品」。
- 混选大贸、跨境、直邮商品点击结算：提示「大贸、跨境、直邮需分开结算」。
- 购物车为空：展示空购物车插画与文案「去逛逛商城吧，选购您心仪的商品～」，保留推荐区。
- 商品失效：进入失效商品区，不可勾选、不可结算，可清空失效商品。
- 推送失败：PC 订单列表展示推送失败状态，并展示重新推送入口。
- 删除购物车商品、清空失效商品、批量删除需二次确认。

### 4.6 权限规则（按需）
- PC 后台业务配置保存权限：仅有系统配置权限的后台账号可操作。
- PC 商品管理批次查看权限：仅有商品管理查看权限的后台账号可查看。
- PC 订单详情、分账信息、订单日志权限：仅有订单管理查看权限的后台账号可查看。
- 小程序购物车能力面向已登录用户；未登录时需先完成登录授权，具体登录链路沿用线上逻辑。

### 4.7 时效与有效期（按需）
- 购物车商品有效性需在展示与提交订单时校验。
- 商品规格有效期展示在选规格、购物车、确认订单、订单详情中。
- 待支付订单超时取消时效沿用业务配置中的待付款订单自动取消时间。
- 售后申请期限、确认收货、评价时效沿用现有订单配置。

### 4.8 数据来源（按需）
- 商品列表：来源于商品中心商品数据。
- 商品批次：来源于 API 商品批次同步数据，包含供应商库位编码、批次号、价格、库存、有效期、商城状态等。
- 购物车：来源于用户购物车数据，包含商品、规格、数量、勾选状态、加购时间。
- 订单列表与订单详情：来源于订单中心订单数据。
- 分账信息：来源于订单分账数据。
- 订单日志：来源于订单操作日志。

### 4.9 通知机制（按需）
- 加入购物车成功：小程序 Toast 提示「已加入购物车」。
- 订单提交成功：小程序 Toast 提示「订单已提交，请及时支付」。
- 支付成功拆单：小程序 Toast 提示「支付成功，多个商品可能拆为多单发货」。
- PC 后台配置保存成功：Toast 提示保存结果和当前加购范围。
- 推送失败不新增主动通知，本期在订单列表状态中展示。

### 4.10 上限与极限（按需）
- 跨境订单支付后拆单：一个子单最多 6 个。
- 购物车单商品数量不得小于 1。
- 规格库存为 0 或规格不可购买时，不允许加入购物车。
- 单次结算仅允许一种商品类型。
- 购物车列表需支持多分组展示，超出屏幕可滚动。

## 5. 状态与异常

### 5.1 业务状态
- 购物车开关：开启 / 关闭。
- 商品状态：可购买 / 库存不足 / 已下架 / 已失效。
- 商品类型：普通商品 / 跨境商品 / 直邮商品。
- 购物车商品状态：正常 / 已勾选 / 未勾选 / 失效。
- 订单状态：待付款 / 待发货 / 已发货 / 已完成 / 已关闭。
- 推送状态：不需要推送 / 推送成功 / 推送失败。
- 管理态：普通结算态 / 批量管理态。

### 5.2 空态
- 订单列表无匹配结果：展示「暂无匹配订单」。
- 购物车为空：展示空购物车插画、空态文案和推荐商品区。
- 我的订单无数据：展示「暂无订单」。

### 5.4 异常态 / 失败态
- 商品规格库存不足：规格置灰，点击提示不可购买。
- 购物车商品失效：进入失效商品区，展示失效标识，不可勾选。
- 混选商品类型：阻断结算并提示分开结算。
- 订单推送失败：PC 订单列表展示「推送失败」和「重新推送」入口。
- 支付后拆单失败：待确认后端返回规则，本期原型不展开，需保留失败日志。

### 5.5 状态-操作矩阵（按需）
| 状态 | 可执行操作 | 备注 |
| --- | --- | --- |
| 购物车开关关闭 | 立即购买 | 不展示加入购物车 |
| 购物车开关开启且类型命中范围 | 加入购物车、立即购买 | 按商品类型配置判断 |
| 商品规格可购买 | 选择规格、加购、立即购买 | 库存需大于 0 |
| 商品规格不可购买 | 查看、切换其他规格 | 点击提示库存不足 |
| 购物车正常商品 | 勾选、改数量、收藏、删除、结算 | 不同类型不可混合结算 |
| 购物车失效商品 | 清空失效商品 | 不可勾选、不可结算 |

## 6. 衍生需求与默认授权项

### 6.1 ID 规则
- 待支付整单生成主单号。
- 子单序号按拆单顺序从 `01` 开始递增。
- 支付后子单号等于主单号。
