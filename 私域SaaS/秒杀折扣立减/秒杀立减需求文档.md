# 秒杀活动 / 折扣立减限购配置需求文档

---

## 1. 业务背景

### 1.1 需求描述

当前「秒杀活动」「折扣立减」已有活动创建、商品设置、活动商品管理能力，但限购配置不完整，无法分别控制：

- 活动维度：活动总可售数量、单用户在活动内可购买数量。
- 商品维度：单商品在活动内总可售数量、单用户在活动内对单商品可购买数量。

运营配置活动时，只能配置活动价、立减金额、折扣等价格信息，缺少总量和用户维度的购买约束，容易出现活动库存被快速消耗、单用户超量购买、商品活动量与商品规格库存不匹配等问题。

### 1.3 目标结果

- 秒杀活动支持活动维度和商品维度限购配置。
- 折扣立减在保留原有「立减 / 打折」活动类型、价格配置和活动商品管理能力的基础上，补充活动维度和商品维度限购配置。
- 创建活动保存后返回列表，运营可从列表进入「活动商品」继续维护商品活动数量、商品用户限购，并支持批量设置。
- 所有限购字段为空时表示不限购；填写时统一校验正整数，避免无效配置进入活动链路。

### 1.4 本期范围

- 菜单范围：仅涉及「营销中心 / 秒杀活动」「营销中心 / 折扣立减」。
- 秒杀活动列表、新建、编辑、活动商品管理。
- 折扣立减列表、新建、编辑、活动商品管理。
- 秒杀活动新增活动维度限购字段：
  - 活动总数量。
  - 活动用户限购。
- 秒杀活动商品设置和活动商品管理新增商品维度限购字段：
  - 商品活动数量。
  - 商品用户限购。
- 折扣立减新增活动维度和商品维度限购字段，规则与秒杀活动一致。
- 商品设置和活动商品管理均支持批量设置商品活动数量、商品用户限购。

### 1.5 本期不做

- 不调整前台活动页面视觉样式。
- 不调整订单支付、退款、售后链路的页面展示，仅要求下单校验接入本次限购规则。

## 2. 关键业务规则

### 4.1 唯一性 / 并发性

- 活动购买扣减需按服务端库存和限购规则做原子校验，避免并发下超卖或突破用户限购。
- 当多个用户同时购买同一活动商品时，需同时校验：
  - 活动剩余总数量。
  - 用户活动剩余可购数量。
  - 商品活动剩余数量。
  - 用户商品剩余可购数量。
- 任一维度不满足时，本次下单不得成功。

### 4.2 字段定义

- 活动总数量：当前活动下所有商品总计可售数量。
  - 是否必填：否。
  - 默认值：空。
  - 范围 / 长度：正整数。
  - 不允许：0、负数、小数、非数字。
  - 为空含义：不限购。

- 活动用户限购：单个用户在当前活动内累计可购买数量。
  - 是否必填：否。
  - 默认值：空。
  - 范围 / 长度：正整数。
  - 不允许：0、负数、小数、非数字。
  - 为空含义：不限购。


- 商品活动数量：单个商品在当前活动内总计可售数量。
  - 是否必填：否。
  - 默认值：空。
  - 范围 / 长度：正整数。
  - 不允许：0、负数、小数、非数字、大于商品规格数量。
  - 为空含义：不限购。

- 商品用户限购：单个用户在当前活动内对单个商品可购买数量。
  - 是否必填：否。
  - 默认值：空。
  - 范围 / 长度：正整数。
  - 不允许：0、负数、小数、非数字、大于商品活动数量。
  - 为空含义：不限购。


### 4.3 计算规则

- 活动总数量校验：
  - 若活动总数量为空，不限制活动整体售卖数量。
  - 若活动总数量不为空，活动内所有商品累计成交数量不得超过活动总数量。

- 活动用户限购校验：
  - 若活动用户限购为空，不限制用户在活动内累计购买数量。
  - 若活动用户限购不为空，同一用户在该活动内历史已购数量 + 本次购买数量不得超过活动用户限购。
  - 活动总数量和活动用户限购均填写时，活动用户限购不得大于活动总数量。

- 商品活动数量校验：
  - 若商品活动数量为空，不限制该商品活动总售卖数量。
  - 若商品活动数量不为空，该商品在当前活动内累计成交数量不得超过商品活动数量。
  - 商品活动数量不得大于商品规格数量。

- 商品用户限购校验：
  - 若商品用户限购为空，不限制用户在该活动内对该商品的购买数量。
  - 若商品用户限购不为空，同一用户在该活动内对该商品历史已购数量 + 本次购买数量不得超过商品用户限购。
  - 商品活动数量和商品用户限购均填写时，商品用户限购不得大于商品活动数量。

### 4.4 关键联动逻辑

- 秒杀活动新建 / 编辑：
  - 商品设置展示秒杀价、商品活动数量、商品用户限购、排序值、售价、商品名称、属性、规格数量、操作。
- 折扣立减新建 / 编辑：
  - 选择立减时，商品设置展示立减金额、减后价格、商品活动数量、商品用户限购。
  - 选择打折时，商品设置展示折扣、折后价格、商品活动数量、商品用户限购。
- 商品设置批量设置：
  - 秒杀活动支持批量设置秒杀价、商品活动数量、商品用户限购。
  - 折扣立减立减模式支持批量设置立减金额、商品活动数量、商品用户限购。
  - 折扣立减打折模式支持批量设置折扣、商品活动数量、商品用户限购。
  - 批量输入为空时，不覆盖商品原字段值。
- 活动商品管理页可继续编辑商品活动数量、商品用户限购，并支持批量设置。

### 4.5 异常处理规则

- 活动总数量格式错误：提示「活动总数量必须为正整数」。
- 活动用户限购格式错误：提示「活动用户限购必须为正整数」。
- 活动用户限购大于活动总数量：提示「活动用户限购不能大于活动总数量」。
- 商品活动数量格式错误：提示「商品活动数量必须为正整数」。
- 商品用户限购格式错误：提示「商品用户限购必须为正整数」。
- 商品活动数量大于规格数量：提示「商品活动数量不能大于规格数量」。
- 商品用户限购大于商品活动数量：提示「商品用户限购不能大于商品活动数量」。

### 4.6 上限与极限（按需）

- 活动总数量、活动用户限购、商品活动数量、商品用户限购最大值建议不超过 99999999。
- 单个活动可添加商品数量上限沿用现有秒杀 / 折扣立减规则。
- 批量设置仅作用于当前活动商品列表中已添加的商品。
- 已成交订单不因后续调小限购数量而回滚；新订单按最新配置校验。
