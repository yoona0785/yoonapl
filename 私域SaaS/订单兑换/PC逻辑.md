## 订单兑换列表

# 页面路径

营销中心 - 订单兑换

---

# 流程状态

> 说明：订单兑换活动按活动时间和人工操作自动流转状态。

- **待开始**：编辑、复制、删除、活动店铺、推广、日志
- **进行中**：编辑、复制、立即结束、活动店铺、推广、日志
- **已结束**：复制、删除、推广、日志

---

# 列表逻辑

## 活动数据

1. 兑换用户数：统计全部活动中兑换成功或人工审核通过后发放奖励的去重用户数，按用户 ID 去重 count；不随列表搜索条件联动。
2. 兑换订单数：统计全部活动中兑换成功或人工审核通过后发放奖励的兑换订单数，按兑换记录 count；不随列表搜索条件联动。
3. 数据更新时间：展示最近统计时间，格式 YYYY.MM.DD HH:mm:ss；点击刷新后重新计算活动数据。

## 表搜索

1. 活动名称：文本输入-模糊匹配，匹配活动名称，支持空值查询。
2. 活动 ID：文本输入-精确匹配，匹配活动 ID，支持空值查询。
3. 活动时间：日期范围选择，匹配活动开始时间和结束时间，支持空值查询。
4. 活动类型：下拉单选，枚举值为全部类型、订单金额固定积分、订单金额比例积分、商品数量固定积分、商品数量比例积分。
5. 活动状态：下拉单选，枚举值为全部状态、待开始、进行中、已结束。

> 多字段间搜索关系：AND（同时满足）。

## 查询表

**排序与分页**：默认按创建时间倒序排列，默认 20 条/每页，支持分页切换。

1. 活动信息
   - 1.1 活动名称：取活动名称，蓝色常规体展示，点击进入订单兑换详情页；详情页复用订单兑换新增/编辑页，只读不可编辑。
   - 1.2 活动 ID：取活动 ID，展示在活动名称下方。
   - 1.3 活动类型：取活动类型，标签样式展示。

2. 活动时间：展示活动开始时间和结束时间；长期有效活动展示长期有效。
3. 兑换用户：展示该活动成功兑换或审核通过后发放奖励的去重用户数，点击进入兑换数据页，并默认带入当前活动名称筛选。
4. 创建时间：展示创建人和创建时间，时间格式 YYYY-MM-DD HH:mm:ss。
5. 更新时间：展示更新人和更新时间，时间格式 YYYY-MM-DD HH:mm:ss。
6. 活动状态：按统一状态标签展示。
   - 6.1 待开始：橙色状态标签。
   - 6.2 进行中：绿色状态标签。
   - 6.3 已结束：灰色状态标签。
7. 操作列：按活动状态和兑换维度展示。
   - 7.1 待开始：编辑、复制、删除、活动店铺、推广、日志。
   - 7.2 进行中：编辑、复制、立即结束、活动店铺、推广、日志。
   - 7.3 已结束：复制、删除、推广、日志。
   - 7.4 活动店铺：仅待开始、进行中，且兑换维度为店铺维度或店铺产品组维度时展示。

# 列表交互

1. **兑换审核**：点击按钮跳转兑换审核列表页，打开独立页签。
2. **公域店铺**：点击按钮跳转公域店铺列表页，打开独立页签。
3. **订单限制**：点击按钮跳转订单限制页，打开独立页签。
4. **新增**：点击按钮跳转订单兑换新增页，打开独立页签。
5. **活动名称**：点击进入订单兑换详情页，页面所有字段只读不可编辑。
6. **兑换用户**：点击进入兑换数据页，默认带活动名称筛选兑换数据。
7. **编辑**：点击进入订单兑换编辑页，回填当前活动配置。编辑页需兼容历史字段，兑换订单范围可读取 orderStartAt、orderCompletedStartAt、orderRangeStartAt、exchangeOrderStartAt、orderRange、exchangeOrderRange 等字段；活动店铺可读取 stores、selectedStores、activityStores、storeRules、shopRules、productGroupRules、storeIds 等字段。
8. **复制**：点击进入订单兑换新增页，回填当前活动配置并复制为新活动。
9. **删除**：点击后展示二次确认弹窗。
   - 9.1 标题：删除。
   - 9.2 内容：是否确认删除。
   - 9.3 确认后删除活动并刷新列表。
10. **立即结束**：点击后展示二次确认弹窗。
   - 10.1 标题：立即结束。
   - 10.2 内容：是否立即结束。
   - 10.3 确认后活动状态更新为已结束。
11. **活动店铺**：点击跳转活动店铺页，携带活动 ID、活动名称和兑换维度。
12. **推广**：点击打开推广弹窗。
   - 12.1 小程序链接与二维码：展示小程序链接、小程序路径、海报图，支持复制链接、下载小程序码、下载海报。
   - 12.2 小程序分享设置：支持填写分享标题、上传分享图和海报，保存后作为活动推广配置。
13. **日志**：点击打开操作日志弹窗，字段包含日志 ID、操作人、操作描述、提示信息、操作时间。

# 表单逻辑

## 基本信息

1. 活动类型：单选组，必填，枚举值为订单金额固定积分、订单金额比例积分、商品数量固定积分、商品数量比例积分；活动类型决定活动积分规则表达。
2. 活动名称：文本输入，必填，限制 50 字符内。
3. 活动时间：日期时间范围，必填或勾选长期有效；未勾选长期有效时需选择开始时间和结束时间，精确到秒。
4. 活动背景图：图片上传，必填，取上传图片作为小程序订单兑换首页顶部背景图，按原图比例等比展示。
5. 活动规则：多行文本，选填；有配置时小程序背景图右侧展示活动规则入口，点击弹窗展示完整活动规则，内容支持滑动查看。
6. 兑换次数：数字输入，选填，不填写表示不限制用户在本活动内兑换订单数量。

## 规则设置

1. 兑换订单范围：日期时间选择，选填，提示为订单完成开始时间；配置后小程序在兑换入口输入框下方、点击兑换按钮上方展示，格式为 YYYY.MM.DD HH:mm:ss后完成的订单；未配置不展示。
2. 兑换维度：单选组，必填，枚举值为统一维度、店铺维度、店铺产品组维度。
3. 选择店铺：按钮，条件展示。
   - 3.1 统一维度：不展示选择店铺按钮，小程序可兑店铺取全部公域店铺。
   - 3.2 店铺维度：展示选择店铺按钮，可选择公域店铺；活动积分按店铺配置。
   - 3.3 店铺产品组维度：展示选择店铺按钮，按产品组维度选择；禁用中的产品组不可选择，没有产品组或产品组均禁用的店铺不可选择。
4. 活动积分：按兑换维度展示。
   - 4.1 统一维度：展示一行规则表达，格式为订单满/订单每满/商品满/商品每满 + 门槛值 + 单位 + 可兑 + 积分值 + 积分。
   - 4.2 店铺维度：展示店铺维度表格，字段为操作、店铺、活动积分；支持批量设置和逐行填写。
   - 4.3 店铺产品组维度：展示平铺表格，字段为操作、店铺、产品组、活动积分；不按店铺分组，不展示商品 ID 字段；支持批量设置和逐行填写。

## 推广分享

1. 数据播报：开关，选填，开启后小程序首页展示兑换播报。
2. 分享设置：开关，选填，开启后可配置小程序右上角页面分享内容。
3. 分享标题：文本输入，选填，限制 50 字符内。
4. 分享图：图片上传，选填，用于微信小程序分享卡片预览。
5. 海报：图片上传，选填，用于推广弹窗海报展示。

# 列表交互

1. **保存**：点击保存按钮。
   - 1.1 校验必填字段，未填写时展示错误提示。
   - 1.2 校验活动时间，同一时间段只能创建一个活动，活动开始时间或结束时间不能与已有活动重复，时间范围不能与已有活动重叠。
   - 1.3 保存时需保留兼容字段：订单范围同步写入 orderStartAt 和 orderCompletedStartAt；活动店铺同步写入 stores、selectedStores、activityStores，避免小程序读取为空。
   - 1.4 校验通过后保存到活动列表，新增后返回订单兑换列表，编辑后更新列表和小程序前端展示数据。
2. **取消**：点击取消按钮返回订单兑换列表；如有未保存内容，展示离开确认。
3. **选择店铺**：点击打开选择店铺弹窗。
   - 3.1 店铺维度弹窗字段为店铺、创建时间；店铺展示平台标签、店铺名称、ID。
   - 3.2 店铺产品组维度弹窗字段为店铺、产品组、创建时间；产品组只展示产品组名称，不展示商品 ID 数量和启用/禁用提示。
   - 3.3 已选择项置灰勾选不可再次选择。
   - 3.4 支持全选全部可选择项，确认后最新添加的店铺或产品组显示在前。
4. **批量设置**：输入批量门槛和批量积分后点击批量设置，将值写入当前配置表所有可编辑行。
5. **删除**：点击表格行删除，删除当前店铺或产品组配置；删除后该项可重新在选择店铺弹窗中选择。

# 列表逻辑

## 表搜索

1. 用户信息：文本输入-精确匹配，匹配用户 ID 或手机号。
2. 用户昵称：文本输入-模糊匹配。
3. 活动名称：下拉单选，枚举值为全部活动和全部订单兑换活动名称。
4. 审核状态：下拉单选，枚举值为全部状态、待审核、通过、驳回。

## 查询表

**排序与分页**：默认按提交时间倒序排列，默认 20 条/每页。

1. 提交信息：展示兑换订单号、下单平台、店铺名称、截图数量、补充说明。
2. 活动信息：展示活动名称、活动 ID、活动类型。

5. 审核时间：展示审核人和审核时间；未审核展示空值。
6. 审核状态：按状态、积分、审核意见顺序展示。
   - 6.1 待审核：状态标签展示待审核，不展示积分。
   - 6.2 通过：状态标签展示通过，展示兑换积分；审核意见有值时展示。
   - 6.3 驳回：状态标签展示驳回，展示审核意见。
7. 操作列：待审核展示审核；全部状态展示日志。

4. **日志**：点击打开操作日志弹窗，字段包含日志 ID、操作人、操作描述、提示信息、操作时间。

# 表单逻辑

## 基础信息

1. 电商平台：单选组，必填，枚举值为淘宝、天猫、京东、抖音、快手、拼多多、小红书、其他。
2. 店铺名称：文本输入，必填，限制 50 字符内。
3. 店铺 ID：文本输入，必填，平台内唯一。

## 产品组配置

1. 产品组名称：文本输入，必填，限制 50 字符内。
2. 商品 ID：多行文本，必填，多个 ID 可换行输入；同一店铺下同一商品 ID 只能归属一个产品组。
3. 默认积分：数字输入，选填，整数；用于订单兑换活动按店铺-产品组维度生成活动积分默认值。
4. 产品组状态：开关，枚举值为启用、禁用；禁用中的产品组在订单兑换选择店铺时不可选择。

# 列表交互

1. **新增产品组**：点击打开新增产品组弹窗，确认后写入产品组列表。
2. **编辑产品组**：点击打开编辑产品组弹窗，回填当前产品组数据。
3. **删除产品组**：点击展示二次确认弹窗；确认后删除产品组，若活动规则已关联该产品组，同步删除相关规则。
4. **状态开关**：点击可启用或禁用产品组；禁用后小程序和订单兑换活动不再使用该产品组参与校验。
5. **保存**：点击保存店铺信息和产品组配置，返回公域店铺列表。
