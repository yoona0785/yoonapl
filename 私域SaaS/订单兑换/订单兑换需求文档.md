# 订单兑换需求文档

## 1. 需求背景

1.1 用户在天猫、京东、抖音、拼多多等外部平台店铺完成下单后，可在私域小程序中输入外部平台订单号兑换积分。

1.2 系统通过外部平台订单 API 查询订单，并按后台配置的订单兑换活动规则进行校验，满足规则则自动发放积分，不满足规则则不发放。

1.3 当 API 查询不到订单时，用户可在小程序提交下单平台、店铺、订单截图、平台账号 ID 截图和补充说明，由运营在 PC 后台进行人工审核，审核通过后发放积分。

1.4 本功能目标是将公域平台店铺会员引流到私域会员体系，并通过积分激励提升用户绑定、回访和转化。

## 2. 业务目标

2.1 支持运营配置订单兑换活动，包括活动基础信息、兑换规则、可兑店铺、店铺产品组积分、推广分享信息。

2.2 支持用户在小程序端输入外部订单号进行兑换，并查看兑换进度与兑换结果。

2.3 支持运营维护公域店铺及产品组，用于按店铺、产品组维度配置差异化积分规则。

2.4 支持兑换审核、兑换数据查询、订单限制、操作日志、活动推广等完整运营闭环。

## 3. 核心业务规则

3.1 每个外部平台订单号只能兑换一次。

3.2 同一时间段只能创建一个订单兑换活动，活动开始时间、结束时间不可与已有活动重复，时间范围不可与已有活动重叠。

3.3 活动状态按时间和人工操作流转：新建活动后未到开始时间为待开始，到开始时间后为进行中，到结束时间或人工立即结束后为已结束。

3.4 兑换订单范围为选填，配置后仅允许订单完成时间大于等于配置时间的订单兑换。

3.5 兑换维度支持统一维度、店铺维度、店铺产品组维度。

3.6 统一维度按活动统一积分规则兑换，不需要选择店铺。

3.7 店铺维度需要选择公域店铺，并按店铺配置活动积分。

3.8 店铺产品组维度需要按店铺产品组选择，仅启用中的产品组可选择；没有产品组或产品组均禁用的店铺不可选择。

3.9 兑换积分支持固定积分和比例积分，活动积分规则表达支持订单满、订单每满、商品满、商品每满。

3.10 API 查询不到订单时进入人工审核流程，审核通过生成兑换成功记录并发放积分，审核驳回生成兑换失败记录并展示审核意见。

# 列表逻辑

## 活动数据

1. 兑换用户数：统计当前活动下成功兑换或审核通过后发放奖励的去重用户数，按用户 ID 去重 count，展示在活动数据卡片。
2. 兑换订单数：统计当前活动下成功兑换或审核通过后发放奖励的订单数，按兑换记录 count。
3. 数据更新时间：展示活动数据最近统计时间，格式 `YYYY.MM.DD HH:mm:ss`，点击「刷新」后重新计算当前活动统计。

## 表搜索

1. 活动名称：文本输入-模糊匹配，匹配活动名称字段，支持空值查询。
2. 活动 ID：文本输入-精确匹配，匹配活动 ID。
3. 活动类型：下拉单选，枚举值为全部、订单金额固定积分、订单金额比例积分、商品数量固定积分、商品数量比例积分。
4. 活动状态：下拉单选，枚举值为全部、待开始、进行中、已结束。
5. 活动时间：日期时间范围，匹配活动开始时间和结束时间。

> 多字段间搜索关系：AND（同时满足）

## 查询表

**排序与分页**：默认按创建时间倒序排列，默认 10 条/每页，支持分页切换。

1. 活动信息（多信息字段）
   - 1.1 活动名称：取活动名称，蓝色常规体展示，点击进入活动详情页；详情页复用订单兑换新增/编辑页，只读不可编辑。
   - 1.2 活动 ID：取活动 ID，展示在活动名称下方。
   - 1.3 活动类型：取活动类型，标签样式展示。

2. 活动时间：展示开始时间和结束时间；长期有效活动展示「长期有效」。

3. 兑换用户：展示兑换用户数，点击进入兑换数据页，并默认按当前活动名称筛选。

4. 兑换订单：展示兑换订单数。

5. 活动状态：枚举值说明。
   - 待开始：橙色状态标签。
   - 进行中：绿色状态标签。
   - 已结束：灰色状态标签。

6. 创建时间：展示创建人和创建时间，时间格式 `YYYY-MM-DD HH:mm:ss`。

7. 更新时间：展示更新人和更新时间，时间格式 `YYYY-MM-DD HH:mm:ss`。

8. 操作列：按活动状态和兑换维度展示操作。
   - 待开始：编辑、复制、删除、活动店铺、推广、日志。
   - 进行中：复制、立即结束、活动店铺、推广、日志。
   - 已结束：复制、推广、日志。
   - 活动店铺：仅活动状态为待开始/进行中，且兑换维度为店铺维度或店铺产品组维度时展示。

# 列表交互

1. **兑换审核**：点击按钮，跳转兑换审核列表页，打开独立页签。

2. **公域店铺**：点击按钮，跳转公域店铺列表页，打开独立页签。

3. **订单限制**：点击按钮，跳转订单限制页，打开独立页签。

4. **新增**：点击按钮，跳转订单兑换新增页，打开独立页签。

5. **活动名称**：点击活动名称，进入订单兑换详情页，页面字段全部只读不可编辑。

6. **兑换用户**：点击兑换用户数，进入兑换数据页，并自动带入当前活动名称作为筛选条件。

7. **删除**：点击删除，展示二次确认弹窗。
   - 标题：删除
   - 内容：是否确认删除
   - 确认后删除该活动，刷新列表。

8. **立即结束**：点击立即结束，展示二次确认弹窗。
   - 标题：立即结束
   - 内容：是否立即结束，结束后用户将不可领取
   - 确认后将活动状态更新为已结束，结束时间更新为当前时间。

9. **活动店铺**：点击后跳转活动店铺页，携带活动 ID、活动名称和兑换维度。

10. **推广**：点击后打开推广弹窗。
    - 小程序链接与二维码：展示小程序链接、小程序路径、海报图，支持复制链接、下载小程序码、下载海报。
    - 小程序分享设置：支持填写分享标题、上传分享图和海报，保存后作为活动推广配置。

11. **日志**：点击后打开操作日志弹窗，字段包含日志 ID、操作人、操作描述、提示信息、操作时间。

# 表单逻辑

## 基本信息

1. 活动类型：单选组，必填，枚举值为订单金额固定积分、订单金额比例积分、商品数量固定积分、商品数量比例积分。

2. 活动名称：文本输入，必填，限制 50 字符内。

3. 活动时间：日期时间范围，选填，支持长期有效；未勾选长期有效时可选择开始时间和结束时间，精确到秒。

4. 活动背景图：图片上传，必填，取上传图片作为小程序订单兑换首页背景图，按原图比例等比展示。

5. 活动规则：多行文本，选填，有配置时小程序背景图右侧展示活动规则入口，点击后弹窗展示完整规则并支持滑动。

6. 积分抵扣比例：文本输入，选填，用于小程序兑换成功弹窗展示，如 `100积分=1元`。

7. 数据播报：开关，选填，开启后小程序首页展示兑换播报。

## 规则设置

8. 兑换积分：单选组，必填，枚举值为固定积分、比例积分。

9. 兑换订单范围：日期时间选择，选填，提示为订单完成开始时间；配置后小程序展示 `YYYY.MM.DD HH:mm:ss后完成的订单`，未配置不展示。

10. 兑换维度：单选组，必填，枚举值为统一维度、店铺维度、店铺产品组维度。

11. 选择店铺：按钮，条件展示。
    - 11.1 统一维度：不展示选择店铺。
    - 11.2 店铺维度：展示选择店铺按钮，可选择公域店铺。
    - 11.3 店铺产品组维度：展示选择店铺按钮，按产品组维度选择，仅启用中的产品组可选择；没有产品组或产品组均禁用的店铺不可选择。

12. 活动积分：按兑换维度展示。
    - 12.1 统一维度：展示一行规则表达，格式为 `订单满/订单每满/商品满/商品每满 + 门槛值 + 单位 + 可兑 + 积分值 + 积分`。
    - 12.2 店铺维度：展示店铺维度表格，字段为操作、店铺、活动积分；支持批量设置和逐行填写。
    - 12.3 店铺产品组维度：展示平铺表格，字段为操作、店铺、产品组、活动积分；不按店铺分组，不展示商品 ID 字段；支持批量设置和逐行填写。

## 推广分享

13. 小程序分享标题：文本输入，选填，限制 50 字符内。

14. 分享图：图片上传，选填，用于微信小程序分享卡片预览。

15. 海报：图片上传，选填，用于推广弹窗海报展示。

# 列表交互

1. **保存**：点击保存按钮。
   - 1.1 校验必填字段，未填写时在字段下方展示错误提示。
   - 1.2 校验活动时间，同一个时间段只能创建一个活动，活动开始时间或结束时间不能与已有活动重复，时间范围不能与已有活动重叠。
   - 1.3 校验通过后保存到活动列表，新增后返回订单兑换列表，编辑后更新列表和活动店铺数据。

2. **取消**：点击取消按钮，返回订单兑换列表；如有未保存内容，展示离开确认。

3. **选择店铺**：点击打开选择店铺弹窗。
   - 3.1 店铺维度弹窗字段为店铺、创建时间；店铺展示平台标签、店铺名称、ID。
   - 3.2 店铺产品组维度弹窗字段为店铺、产品组、创建时间；产品组只展示产品组名称，不展示商品 ID 数量和启用/禁用提示。
   - 3.3 已选择项置灰勾选不可再次选择。
   - 3.4 支持全选全部可选择项，确认后最新添加的店铺或产品组显示在前。

4. **批量设置**：输入批量门槛和批量积分后点击批量设置，将值写入当前配置表所有可编辑行。

5. **删除**：点击表格行删除，删除当前店铺或产品组配置；删除后该项可重新在选择店铺弹窗中选择。

# 页面逻辑

## 页面状态

> 说明：订单兑换小程序页面展示后台当前进行中的订单兑换活动。

- **进行中**：展示后台配置的活动背景图、数据播报、活动规则入口、兑换区域、兑换流程，用户可输入订单号兑换。

- **无进行中活动**：展示暂无进行中的订单兑换活动，隐藏兑换入口、数据播报、可兑店铺与兑换流程操作。

- **兑换成功**：打开兑换成功弹窗，展示兑换奖励。

- **审核中**：接口查询不到订单时，用户提交订单信息后生成审核记录，在兑换记录中展示审核中。

- **兑换失败**：订单不满足兑换条件或后台审核驳回时展示兑换失败。

## 页面内容

1. 导航栏：二级页导航栏，标题为订单兑换，左侧展示返回按钮，右侧展示小程序胶囊。

2. 活动背景图：取后台订单兑换活动配置的活动背景图。

1. 图片按后台上传尺寸等比展示，宽度铺满小程序页面。

2. 数据播报浮在背景图上，位置为左侧 10px，距离背景图底部 76px，尺寸 106px × 20px。

3. 活动规则浮标展示在背景图右侧，位置 355px、96px，尺寸 20px × 67px，圆角 `6px 0 0 6px`，背景色 `#000000` 20% 透明度，文字白色。

3. 数据播报：后台配置勾选时展示。

1. 数据来源为兑换成功记录。

2. 按奖励发放时间倒序展示，最新发放在前。

3. 播报内容为用户昵称脱敏和兑换积分，格式为 `前2位***后2位 兑换xx积分`。

4. 数据上下滚动展示，一条数据也循环滚动。

4. 兑换区域：展示在背景图层上方，顶部距离数据播报底部 14px。

1. 可兑订单：后台配置兑换订单范围时展示，格式为 `YYYY.MM.DD HH:mm:ss后完成的订单`；未配置不展示。

2. 兑换入口：文本输入框，提示文案为请输入指定店铺订单号；支持复制粘贴。

3. 点击兑换：主按钮，文案为点击兑换。

4. 兑换记录：点击进入兑换记录页。

5. 兑换店铺：展示后台兑换维度关联店铺名称。

- 统一维度：取全部公域店铺名称。

- 店铺维度：取活动中添加的店铺名称。

- 店铺产品组维度：取活动中添加的店铺名称，按店铺添加顺序去重展示。

- 最多展示 5 个店铺，超过 5 个展示 `>>`。

5. 兑换流程：固定展示 4 个步骤。

1. 进入平台查看订单，平台展示淘宝、天猫、京东、拼多多、小红书、抖音、其他。

2. 进入订单详情，查看订单号并复制。

3. 复制单号，回到活动页点击立即兑换。

4. 兑换成功后领取奖励，并可在兑换记录查看进度。

6. 活动规则弹窗：点击背景图右侧活动规则浮标打开。

1. 内容取后台配置的活动规则。

2. 内容区支持上下滑动查看全部。

3. 后台未配置活动规则时，不展示活动规则浮标。

7. 可兑店铺弹窗：点击可兑店铺打开。

1. 展示当前活动可兑店铺名称列表。

2. 店铺来源按兑换维度取值。

# 页面交互

## 页面交互

1. 点击兑换：点击主按钮后校验订单号。

1. 订单号为空时 Toast 提示 `请输入单号进行兑换`。

2. 若订单已兑换，打开提示弹窗，文案为 `订单已兑换，请查看兑换记录`，点击查看记录进入兑换记录页。

3. 若订单命中订单限制或不满足兑换条件，打开失败提示弹窗，文案为 `订单不满足兑换条件，请重新输入`。

4. 若接口查询不到订单，打开信息提交弹窗。

5. 若兑换成功，写入兑换记录并打开兑换成功弹窗。

2. 订单兑换条件校验：点击兑换时执行。

1. 每个订单只能兑换一次。

2. 若后台配置兑换订单范围，则订单完成时间必须大于等于后台配置时间。

3. 店铺维度活动需校验订单所属店铺是否在活动添加店铺内。

4. 店铺产品组维度活动需校验订单所属店铺和商品 ID 是否归属活动添加的启用产品组。

3. 信息提交弹窗：接口查询不到订单时打开。

1. 下单平台：单选，必填，枚举值为淘宝、天猫、京东、抖音、快手、拼多多、小红书、其他。

2. 店铺名称：单选，必填，按所选平台筛选当前活动可兑店铺。

3. 截图：图片上传，必填，至少上传两张，分别为订单详情图和平台账号 ID 图。

4. 补充说明：多行文本，选填，限制 200 字符内。

5. 提交：校验通过后写入后台兑换审核列表，同时生成一条审核中兑换记录。

4. 兑换成功弹窗：兑换成功后打开。

1. 标题为兑换成功。

2. 展示文案 `恭喜您获得以下奖励`。

3. 展示积分奖励和积分抵扣比例。

4. 温馨提示：积分直接发放到注册账户，进入个人中心可查看积分明细。

5. 点击查看奖励进入积分明细页。

5. 兑换失败弹窗：兑换失败时打开。

1. 展示失败原因。

2. 点击好的，知道了关闭弹窗。
