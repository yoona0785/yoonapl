订单兑换小程序 PRD

页面路径

小程序 - 订单兑换

页面逻辑

页面状态

说明：订单兑换小程序页面展示后台当前进行中的订单兑换活动。

- 进行中：展示后台配置的活动背景图、数据播报、活动规则入口、兑换区域、兑换流程，用户可输入订单号兑换。

- 无进行中活动：展示暂无进行中的订单兑换活动，隐藏兑换入口、数据播报、可兑店铺与兑换流程操作。

- 兑换成功：打开兑换成功弹窗，展示兑换奖励。

- 审核中：接口查询不到订单时，用户提交订单信息后生成审核记录，在兑换记录中展示审核中。

- 兑换失败：订单不满足兑换条件或后台审核驳回时展示兑换失败。

页面内容

1. 导航栏：二级页导航栏，标题为订单兑换，左侧展示返回按钮，右侧展示小程序胶囊。

2. 活动背景图：取后台订单兑换活动配置的活动背景图。

1. 图片按后台上传尺寸等比展示，宽度铺满小程序页面。

2. 数据播报浮在背景图上，位置为左侧 10px，距离背景图底部 76px，尺寸 106px × 20px。

3. 活动规则浮标展示在背景图右侧，位置 355px、96px，尺寸 20px × 67px，圆角 6px 0 0 6px，背景色 #000000 20% 透明度，文字白色。

3. 数据播报：后台配置勾选时展示。

1. 数据来源为兑换成功记录。

2. 按奖励发放时间倒序展示，最新发放在前。

3. 播报内容为用户昵称脱敏和兑换积分，格式为 前2位***后2位 兑换xx积分。

4. 数据上下滚动展示，一条数据也循环滚动。

4. 兑换区域：展示在背景图层上方，顶部距离数据播报底部 14px。

1. 可兑订单：后台配置兑换订单范围时展示，格式为 YYYY.MM.DD HH:mm:ss后完成的订单；未配置不展示。

2. 兑换入口：文本输入框，提示文案为请输入指定店铺订单号；支持复制粘贴。

3. 点击兑换：主按钮，文案为点击兑换。

4. 兑换记录：点击进入兑换记录页。

5. 兑换店铺：展示后台兑换维度关联店铺名称。

- 统一维度：取全部公域店铺名称。

- 店铺维度：取活动中添加的店铺名称。

- 店铺产品组维度：取活动中添加的店铺名称，按店铺添加顺序去重展示。

- 最多展示 5 个店铺，超过 5 个展示 >>。

5. 兑换流程：固定展示 4 个步骤。

1. 进入平台查看订单，平台展示淘宝、天猫、京东、拼多多、小红书、抖音、其他。

2. 进入订单详情，查看订单号并复制。

3. 复制单号，回到活动页点击立即兑换。

4. 兑换成功后领取奖励，并可在兑换记录查看进度。

6. 活动规则弹窗：点击背景图右侧活动规则浮标打开。

1. 内容取后台配置的活动规则。

2. 内容区支持上下滑动查看全部。

3. 后台未配置活动规则时，不展示活动规则浮标。

7. 可兑店铺弹窗：点击可兑店铺打开。

1. 展示当前活动可兑店铺名称列表。

2. 店铺来源按兑换维度取值。

页面交互

页面交互

1. 点击兑换：点击主按钮后校验订单号。

1. 订单号为空时 Toast 提示 请输入单号进行兑换。

2. 若订单已兑换，打开提示弹窗，文案为 订单已兑换，请查看兑换记录，点击查看记录进入兑换记录页。

3. 若订单命中订单限制或不满足兑换条件，打开失败提示弹窗，文案为 订单不满足兑换条件，请重新输入。

4. 若接口查询不到订单，打开信息提交弹窗。

5. 若兑换成功，写入兑换记录并打开兑换成功弹窗。

2. 订单兑换条件校验：点击兑换时执行。

1. 每个订单只能兑换一次。

2. 若后台配置兑换订单范围，则订单完成时间必须大于等于后台配置时间。

3. 店铺维度活动需校验订单所属店铺是否在活动添加店铺内。

4. 店铺产品组维度活动需校验订单所属店铺和商品 ID 是否归属活动添加的启用产品组。

3. 信息提交弹窗：接口查询不到订单时打开。

1. 下单平台：单选，必填，枚举值为淘宝、天猫、京东、抖音、快手、拼多多、小红书、其他。

2. 店铺名称：单选，必填，按所选平台筛选当前活动可兑店铺。

3. 截图：图片上传，必填，至少上传两张，分别为订单详情图和平台账号 ID 图。

4. 补充说明：多行文本，选填，限制 200 字符内。

5. 提交：校验通过后写入后台兑换审核列表，同时生成一条审核中兑换记录。

4. 兑换成功弹窗：兑换成功后打开。

1. 标题为兑换成功。

2. 展示文案 恭喜您获得以下奖励。

3. 展示积分奖励和积分抵扣比例。

4. 温馨提示：积分直接发放到注册账户，进入个人中心可查看积分明细。

5. 点击查看奖励进入积分明细页。

5. 兑换失败弹窗：兑换失败时打开。

1. 展示失败原因。

2. 点击好的，知道了关闭弹窗。

页面路径

小程序 - 订单兑换 - 兑换记录

页面逻辑

页面状态

说明：兑换记录按兑换结果分为全部、兑换成功、兑换失败三种筛选状态。

- 全部：展示兑换成功、审核中、兑换失败记录。

- 兑换成功：仅展示自动兑换成功和手动审核通过后发放积分的记录。

- 兑换失败：仅展示后台审核驳回的失败记录。

- 空状态：当前筛选条件下无记录时，展示空状态插画、提示文案和操作按钮。

页面内容

1. 导航栏：二级页导航栏，标题为兑换记录，左侧展示返回按钮，右侧展示小程序胶囊。

2. 标签筛选：展示全部、兑换成功、兑换失败。

1. 点击标签后切换当前记录列表。

2. 标签区域不展示整条白底，仅展示标签按钮。

3. 数据记录：按记录时间倒序展示。

1. 左侧订单信息：展示兑换订单号，格式为 兑换订单{订单号}。

2. 右侧成功信息：兑换成功时展示 +xx积分 和积分发放时间。

3. 右侧审核中信息：审核中时展示 审核中 和提交时间。

4. 右侧失败信息：兑换失败时展示 兑换失败 和后台填写的失败原因。

4. 空状态：无记录时展示。

1. 展示空状态插画。

2. 提示文案为暂无积分记录，去商城逛逛吧。

3. 操作按钮为去商城、去首页。

页面交互

页面交互

1. 全部：点击后展示全部记录。

2. 兑换成功：点击后筛选 status = 兑换成功 的记录。

3. 兑换失败：点击后筛选 status = 兑换失败 的记录。

4. 去商城：点击后跳转商城页面。

5. 去首页：点击后返回订单兑换首页。

6. 返回：点击导航栏返回按钮，返回订单兑换首页。
